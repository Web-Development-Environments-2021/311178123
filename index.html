the web is Read-Only and static.

the web is wildly Read-Write.

the web is portable and personal - Read-Write intelligent web.

search engines retrieve macro contents. Search is very fast but many times results are inaccurate or more than users can chew.

search engines retrieve tags with micro contents. The process of tagging is manual, tedious and covers negligible percents of the WWW.

search engines will retrieve micro content texts which were tagged automatically.This implies translating billions of Web 1.0 macro contents into micro contents. The result could be more precise search because tagging can solve part of the ambiguity that homonyms and synonyms introduce into the process of Search.

Companies publish content that people consume (e.g. Ynet)

People publish content that other people can consume, companies build platforms that let people publish content for other people (e.g. YouTube, Wikipedia)

People build applications that people can interact with, companies build platforms that let people publish services by leveraging the associations between people or special content ( e. g. Facebook, Instagram)




Use Case example - noa kirel
  • Web 1.0 - for finding the relevant data we want about "noa kirel - latest song", the user have to write a specific URL.
  • Web 2.0 - for finding result we can use "noa kirel - latest song" as a query to a search engines and we will get result from the query as web tags.
  • Web 3.0 - for finding result we can use "noa kirel - latest song" as a query to a search engines and we will get result that contain the query words and semantic words as tags or web info.
