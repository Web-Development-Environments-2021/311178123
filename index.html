The web is static and users can only read the page content. (Ynet, Walla etc)

Users can write to the web page in addition to reading - blogs. (Flickr, MySpace, Wikipedia)

The "Semantic Web", The Read-Write operations are more intelligent and content aware (AI is introduced).

search engines retrieve macro contents. Search is very fast but many times results are inaccurate or more than users can chew.

search engines retrieve tags with micro contents. The process of tagging is manual, tedious and covers negligible percents of the WWW.

search engines will retrieve micro content texts which were tagged automatically.This implies translating billions of Web 1.0 macro contents into micro contents. The result could be more precise search because tagging can solve part of the ambiguity that homonyms and synonyms introduce into the process of Search.

Use Case example - Stuxnet
  • Web 1.0 - "Stuxnet Iran attack" query: the user have to write a specific URL of the website (no tagging).
  • Web 2.0 - "Stuxnet Iran attack" query: search engines get result from the query as web tags (Videos, blogs, audios) which may result in information the user does not need.
  • Web 3.0 - "Stuxnet Iran attack" query: search engines get result that contain the query words and other semantic words.
